## 3.2.3 AI Voicebot for Customer Support (RAG System)

Rahul also contributed to building an **AI-powered voice assistant for customer support** using Retrieval-Augmented Generation architecture.

The goal of the system was to allow customers to interact with automated voice support that could retrieve relevant information and provide contextual responses.

---

### Architecture Components

The system was built using multiple modern AI and infrastructure tools.

#### Voice Infrastructure

**LiveKit**

LiveKit was used for real-time voice communication infrastructure that enables streaming audio interaction between users and backend services.

---

#### Knowledge Retrieval System

**Vector Databases**

The system used vector databases such as:

- Qdrant
- Pinecone

These databases store embeddings of documents and allow semantic retrieval of relevant information based on user queries.

---

#### Large Language Model Integration

**Google Gemini API**

The system used Google's Gemini model to generate responses based on retrieved context.

The LLM received:

- user query
- retrieved documents from vector database
- system prompts

and generated contextual responses.

---

#### Backend APIs

**FastAPI**

FastAPI was used to build the backend service layer responsible for:

- handling requests
- orchestrating RAG pipelines
- connecting vector database retrieval
- invoking LLM responses
- returning responses to the voice interface

---

### RAG Pipeline Flow

The high-level pipeline followed this sequence:

1. User speaks a query through voice interface
2. Speech is converted to text
3. Query embedding is generated
4. Vector database retrieves relevant knowledge chunks
5. Retrieved context is injected into LLM prompt
6. LLM generates contextual answer
7. Response is converted to speech and returned to the user

This architecture allowed the voice assistant to provide accurate answers using internal knowledge sources.
